MotrixArena S1 赛事资料提交须知 (TXT版)
====================================================================

一、 提交总览
参赛队伍最终需提交一个统一打包的总压缩包，命名严格要求为："MotrixArena_S1_{队伍标号}.zip"[1]。
文件完整性检查要求提交包内必须包含且仅包含以下三个文件，请勿包含任何临时文件、系统缓存（如 __pycache__）或其他无关资料[2]：
1. 模型文件包：runs_{队伍标号}.zip
2. 代码包：code_{队伍标号}.zip
3. 技术说明文档：技术说明文档_{队伍标号}.pdf [1]。

--------------------------------------------------------------------
二、 完整目录结构规范
--------------------------------------------------------------------
MotrixArena_S1_{队伍标号}.zip
├── runs_{队伍标号}.zip
│   ├── MotrixArena_S1_section001_{队伍标号}/
│   │   └── (自动生成的PPO时间戳目录)/
│   │       └── checkpoints/
│   │           └── best_agent.pickle
│   ├── MotrixArena_S1_section011_{队伍标号}/...
│   ├── MotrixArena_S1_section012_{队伍标号}/...
│   └── MotrixArena_S1_section013_{队伍标号}/...
├── code_{队伍标号}.zip
│   ├── vbot_{队伍标号}/ (或 .zip)
│   │   ├── cfg.py
│   │   └── *_np.py 场景代码
│   ├── cfgs_{队伍标号}.py
│   └── ReadMe_{队伍标号}.md (或 .txt)
└── 技术说明文档_{队伍标号}.pdf [1-4]

--------------------------------------------------------------------
三、 各模块详细要求
--------------------------------------------------------------------
【1】 模型文件包 (runs_{队伍标号}.zip) [2]
- 内部必须按功能分类存放4个赛段的模型文件。若使用单个模型跑越障全过程，文件夹命名为：MotrixArena_S1_section01_{队伍标号}[2]。
- 核心红线：每个赛段的 checkpoints 文件夹下，有且仅能保留一个 best_agent.pickle 文件！必须手动删除多余的 agent_xxx.pickle 等中间训练记录[2-4]。

【2】 代码包 (code_{队伍标号}.zip) [4]
- 代码必须规范，关键函数/类需有详细注释（明确功能、入参、出参）[4]。
- 必须确保评委解压后可使用官方统一的 play.py 直接运行[4, 5]。
- vbot_{队伍标号} 文件夹：必须包含 cfg.py，且配置好四个赛段的环境参数；对应的 _np.py 文件需要详细注释[6]。
- 允许修改范围：vbot中的 cfg.py、对应的 Python 环境文件 (*_np.py)、全局配置文件 cfgs.py[5]。
- 严格限制：非必要不要新增其他运行环境依赖；非必要不要修改底层 PPO 算法！如确有必要，必须向赛事组反馈并确保能独立运行[5, 7]。

【3】 技术说明文档 (技术说明文档_{队伍标号}.pdf) [1, 5]
- 必须包含的章节：
  1. 文档封面：赛道名称、团队名称、提交日期[5]。
  2. 目录：清晰列出章节及页码[5]。
  3. 环境说明：基础操作系统，修改说明[8]。
  4. 快速运行指南：修改步骤与具体的运行指令[8]。
  5. 技术细节与结果：详细阐述设计逻辑与核心评审要点[8]。
- 核心评审要点（必须在文档中重点体现）：
  1. 随机起始能力：验证模型的泛化能力，不依赖固定起点[9]。
  2. 防失败机制：保持稳定“不摔倒 (Stability)”，且“不出界 (Boundary)”[9]。

[ ] code 包结构完整，包含 vbot、cfgs.py、ReadMe，且已添加规范注释。
[ ] 未新增非必要依赖，未私自修改底层 PPO 算法。
[ ] 文档包含封面、目录及所有必填章节，并凸显了随机起始和防失败机制。
[ ] 强烈建议：已清理 __pycache__ 等所有冗余文件，并在原始例程上成功完成了最终测试[7, 9]。
