## 第一章 总体思路

### 1.1 训练简述

#### 1.1.1 训练方式

本方案采用**端到端的深度强化学习（Deep Reinforcement Learning, DRL）**方法完成四足机器狗 VBot 的全场景导航任务。为使模型在不同复杂度、不同地形条件下可靠收敛，我们将训练过程划分为 **四个独立赛段** 进行分阶段训练：

| 赛段 | 环境名称 | 地形特征 | 训练目标 |
|------|----------|---------|---------|
| Section 001 | `vbot_navigation_section001` | 圆形平地赛场 | 基础步态+泛化导航 |
| Section 011 | `vbot_navigation_section011` | 坑洼+斜坡+2026平台 | 障碍避让+红包收集 |
| Section 012 | `vbot_navigation_section012` | 波浪地形+吊桥/河床+楼梯 | 路线决策+复杂地形穿越 |
| Section 013 | `vbot_navigation_section013` | 滚动金球+随机地形 | 鲁棒抗冲击+终点庆祝 |

每个赛段通过**针对性的环境物理约束**与**动态奖励重塑（Reward Shaping）**，引导策略网络在各赛段演化出差异化的最优行为。此外，Section 011 支持从 Section 001 的预训练权重进行**课程学习（Curriculum Learning）**迁移，Section 012 同样支持从 Section 011 权重继续训练，实现知识的逐级传递。

所有训练均使用 2048 个并行仿真环境，总训练步数约 61M steps（`1024 × 60,000`），利用 GPU 加速的 PPO 算法实现高效并行采样。

#### 1.1.2 模型结构

严格遵守赛事组**"非必要不要修改底层 PPO 算法"**的规定，本方案的训练基座完全沿用 MotrixLab 官方提供的标准 **PPO（Proximal Policy Optimization）**算法架构。

**Actor-Critic 网络结构：**

```
策略网络 (Policy Network):  Input → [256] → [128] → [64] → Output
价值网络 (Value Network):   Input → [256] → [128] → [64] → Output
```

四个赛段的网络结构**完全统一**（均为 `(256, 128, 64)` 三层隐藏层），以支持跨赛段的课程学习权重迁移。我们未对底层网络做任何破坏性修改，而是将全部优化重心放在了**环境配置（`cfg.py`）**以及**针对不同赛段的物理约束与奖励函数设计**上。

#### 1.1.3 关键超参

**PPO 算法核心超参数（四赛段统一）：**

| 参数 | 值 | 说明 |
|------|----|----|
| `learning_rate` | 3e-4 | Adam 优化器学习率 |
| `discount_factor` | 0.99 | 折扣因子，平衡短期/长期回报 |
| `lambda_param` | 0.95 | GAE λ，偏差-方差权衡 |
| `rollouts` | 48 | 每次更新前采集的步数 |
| `learning_epochs` | 6 | 每批数据重复训练轮数 |
| `mini_batches` | 32 | Mini-batch 数量 |
| `ratio_clip` / `value_clip` | 0.2 | PPO 策略/价值裁剪系数 |
| `grad_norm_clip` | 1.0 | 梯度裁剪阈值 |
| `num_envs` | 2048 | 并行训练环境数 |
| `check_point_interval` | 500 | 保存模型间隔（epoch） |

**PD 控制器物理参数（四赛段统一）：**

| 参数 | 值 | 物理含义 |
|------|----|----|
| `stiffness (Kp)` | 60 N·m/rad | 关节位置恢复的核心刚性 |
| `damping (Kd)` | 0.8 N·m·s/rad | 吸收冲击瞬时能量，抑制高频震荡 |
| `action_scale` | 0.25 | 策略输出→目标角度映射系数 |
| `sim_dt` / `ctrl_dt` | 0.01s (100Hz) | 仿真与控制同频步长 |
| 有效 PD 力矩上限 | 15 N·m | Kp × action_scale = 60 × 0.25 |

**关键防失败参数（逐赛段差异化）：**

| 参数 | Section 001 | Section 011 | Section 012 | Section 013 |
|------|------------|------------|------------|------------|
| 倾斜角终止阈值 | **30°** | 45° | 45° | 45° |
| 高度跌落比例 | 0.18m绝对值 | 40%目标高度 | 40%目标高度 | 40%目标高度 |
| 宽限期步数 | 20 步 | 10 步 | 10 步 | 10 步 |
| 终止惩罚 | **-10.0** | **-50.0** | **-50.0** | **-200.0** |
| 最大 Episode 时长 | 40s / 4000步 | 40s / 4000步 | **120s / 12000步** | 40s / 4000步 |

> **设计理念：** Section 001（平地）采用更严格的姿态阈值（30°）和极轻的终止惩罚（-10），注重基础步态质量。Section 011/012 将终止惩罚降至 -50 以**鼓励冒险探索复杂地形**。Section 013 回调至 -200 的重罚，**严守抗冲击场景下的生存底线**。

#### 1.1.4 奖励简述

我们在基础的进度奖励和惩罚之上（如基础的 `termination: -200.0`、`arrival_bonus: 10.0` 等），进行了大量定制化的奖励重塑。核心设计原则如下：

**1. 破除"安全港效应"：**
- 在 Section 012 中，将站立高度奖励权重 `HEIGHT_REWARD_SCALE` 从 2.0 降至 **0.5**
- 消除"站着不走就能骗分"的局部最优陷阱
- 同时放宽 `HEIGHT_REWARD_SIGMA` 至 0.25，防止过障时高度波动导致奖励信号归零

**2. 强化探索动力：**
- Section 012 的前进速度权重 `FORWARD_VEL_SCALE` 提升至 **3.0**
- Y 轴进度奖励 `PROGRESS_SCALE` 提升至 **4.0**
- Section 001 新增 `standstill_penalty = -1.0`，直接惩罚原地不动

**3. 策略引导奖励：**
- Section 013 滚球区设定 `BALL_SURVIVE_SHAPING = 2.0` 的即时奖励
- 鼓励触球后维持站立的"策略B"鲁棒行为，而非保守绕行
- 吊桥/河床路线设计不同分值权重，体现风险-回报正相关

**4. 密集导航引导：**
- 各赛段均设置 Y 轴检查点（`CHECKPOINTS_Y`）
- 通过 `CHECKPOINT_SHAPING` 提供中间里程碑的稠密奖励信号
- 距离塑形函数 `exp(-dist/σ)` 保证全局梯度连续

### 1.2 第一赛段训练策略（Section 001 — 平地导航）

#### 1.2.1 整体训练思路

Section 001 为**圆形平地赛场基础导航任务**。核心思路是构建机器狗基础的**平地步态与泛化追踪能力**，作为后续越障赛段的预训练基座。

为满足赛事对**"随机起始能力"**的一票否决要求，我们在训练中引入了极强的**领域随机化（Domain Randomization）**：

- **初始位置随机化：** X/Y 轴各允许 ±0.5m 的随机偏移（相对于中心位置 `[0.0, -2.4, 0.5]`）
- **目标位置：** 固定在 `Y=10.2`，起始与目标距离约 12.6m
- **圆形边界：** 半径 10.0m 的圆形赛场，越界即终止

模型根据实时的**相对坐标**进行动态轨迹规划，而非死记固定路线。设计了**两阶段触发点系统**（内圈 trigger_a + 圆心 trigger_b），引导策略网络学习逐步逼近目标的稳健路径。

**训练流程图：**

```
初始化机器狗 → 随机位置与朝向
     ↓
本体感知观测 (54维) → 策略网络 (256,128,64)
     ↓
输出12维关节动作 → PD力矩控制 (Kp=60, Kd=0.8)
     ↓
环境仿真100Hz → 摔倒/越界检测
     ↓
├─ 成功到达 trigger_a → +10分 → 继续到 trigger_b → +10分
└─ 摔倒/越界 → 终止 + 惩罚(-10.0)
```

#### 1.2.2 终止条件

Section 001 的终止条件设计最为严格（30°倾斜阈值），以确保基础步态的高质量：

| 终止条件 | 阈值 | 说明 |
|---------|------|------|
| 躯体碰撞 | 接触传感器触发 | 躯干/大腿/小腿触地 |
| 姿态失控 | Roll/Pitch > **30°** | 比越障赛段更严格 |
| 高度跌落 | 基座高度 < **0.18m** | 绝对高度阈值 |
| 越界 | 距圆心 > **10m** | 圆形边界检测 |
| 关节异常 | DOF速度 > 100 或 NaN/Inf | 物理仿真异常检测 |
| 重置宽限期 | 前 **20 步** 不终止 | 避免重置瞬间误判 |

#### 1.2.3 数据处理与观测量

**观测空间：54 维**（与后续赛段课程学习兼容）

| 观测分量 | 维度 | 归一化系数 | 说明 |
|---------|------|-----------|------|
| 基座线速度 `base_linvel` | 3 | ÷2.0 | 三轴运动速度反馈 |
| 基座角速度 `base_gyro` | 3 | ÷0.25 | 陀螺仪角速度 |
| 投影重力向量 | 3 | — | 用于判断躯干倾斜方向 |
| 关节角度偏差 | 12 | ÷1.0 | 当前角度 - 默认站立角度 |
| 关节速度 | 12 | ×0.05 | 12 个关节的角速度 |
| 上一步动作 | 12 | — | 动作记忆，平滑控制信号 |
| 目标指令 | 3 | ÷[2.0, 2.0, 0.25] | 归一化后的线速度+角速度指令 |
| 位置误差 | 2 | ÷5.0 | 相对目标的 XY 偏差 |
| 朝向误差 | 1 | ÷π | 当前朝向与目标方向的角差 |
| 归一化距离 | 1 | exp(-d/σ) | 距目标距离的密集信号 |
| 到达标志 | 1 | 0/1 | 是否已到达目标 |
| 停稳就绪 | 1 | 0/1 | 速度是否足够低 |

**动作空间：12 维** — 12 个关节的目标角度偏移量，通过 PD 控制器转换为力矩：

$$T_{joint} = K_p \times (a_{scale} \times action + \theta_{default} - \theta_{current}) - K_d \times \dot{\theta}_{current}$$

其中 $K_p = 60$，$K_d = 0.8$，$a_{scale} = 0.25$。力矩限幅为 `[17, 17, 34] × 4`（髋关节17, 大腿17, 小腿34，每条腿分别限制）。

#### 1.2.4 奖励设计细节

Section 001 的奖励设计重点解决了**"站原地不动即可获得高奖励"**的经典问题：

| 奖励组件 | 权重/规模 | 公式/逻辑 |
|---------|-----------|----------|
| 站立高度 | 0.5 × exp(-err²/0.08) | **刻意降低 4 倍**，仅作基线 |
| 存活奖励 | +0.1（站立）/ 0（趴下） | 降低以避免安全港 |
| **进度奖励** | clip(Δd × 5.0, -1, 2) | **核心信号**，大幅提升权重 |
| 距离塑形 | exp(-dist/3.0) × 1.0 | 密集距离衰减信号 |
| 阶段完成 | +10.0（一次性） | trigger_a / trigger_b 各加分 |
| 速度跟踪 | exp(-err/0.25) × 1.5 | 跟踪目标线速度 |
| **静止惩罚** | **-1.0** | 防止站着不动的关键设计 |
| 朝向对齐 | exp(-\|err\|/0.5) × 0.5 | 引导面向目标 |
| 步态奖励 | ×1.0 | 足部腾空时间激励 |

#### 1.2.5 其他处理

**观测噪声注入（提升 Sim-to-Real 鲁棒性）：**

| 噪声源 | 缩放系数 | 设计意图 |
|--------|---------|---------|
| 关节角度 | 0.03 | 轻微位置噪声 |
| **关节速度** | **1.5** | **主要随机扰动源**，强制模型学习滤波 |
| 陀螺仪 | 0.2 | 模拟传感器误差 |
| 重力向量 | 0.05 | 模拟姿态估计偏差 |
| 线速度 | 0.1 | 模拟速度估计噪声 |

**动作平滑处理：** 使用一阶低通滤波器（`action_filter_alpha = 0.3`）：

$$a_{smooth} = \alpha \times a_{new} + (1-\alpha) \times a_{old}$$

**随机推力扰动：** 训练中以 `random_push_scale = 0.3` 的概率施加外力，提升策略的抗扰动能力。

### 1.3 第二赛段训练策略（Section 011 — 障碍导航）

#### 1.3.1 整体训练思路

Section 011 为**障碍导航赛段**，机器狗需要从起始位置出发，穿越坑洼地形中的笑脸障碍物，攀爬斜坡收集红包，最终抵达"2026"目标平台。

核心训练策略为**"生存优先 + 稠密引导"**模式：

- **课程学习支持：** 可通过 `curriculum_from_001=True` 加载 Section 001 预训练权重，此时观测空间锁定为 54 维（与 001 完全一致），PD 力矩不限幅，实现知识的无损迁移
- **崎岖地形模式：** 可通过 `rough_terrain_mode=True` 扩展观测空间至 `54 + 17 + N×27` 维，增加足部接触力、基座高度、状态历史等感知信息，增强地形适应能力
- **终止惩罚宽容化：** 将失败惩罚从默认 -200 降至 **-50**，降低早期探索阶段的"保守收敛"倾向

**赛道元素：**
- 3 个笑脸障碍物（触发半径 1.0m）
- 3 个红包收集点（触发半径 1.0m）
- 终点区域："2026"平台（中心 Y=7.83，半径 1.0m）
- 庆祝动作触发

#### 1.3.2 终止条件

| 终止条件 | 阈值 | 说明 |
|---------|------|------|
| 基座接触地面 | `base_contact` 传感器触发 | 躯干碰地判定 |
| 姿态失控 | Roll/Pitch > **45°** | 比 001 放宽，适应坑洼地形 |
| 越界 | 超出矩形边界（默认 X:±20, Y:-10~20） | 赛场范围约束 |
| 高度跌落 | 基座高度 < 40% × 目标高度 | 比例式判定 |
| 关节异常 | DOF速度 > 100 或 NaN/Inf | 物理仿真保护 |
| 宽限期 | 前 **10 步** 不终止 | 避免重置瞬间误判 |

#### 1.3.3 数据处理与观测量

Section 011 支持 **三种观测模式**，通过配置参数切换：

**模式一 — 课程学习模式（`curriculum_from_001=True`）：54 维**
- 与 Section 001 完全一致的观测向量
- 用于加载 001 预训练权重进行知识迁移
- PD 力矩不限幅（与 001 一致）

**模式二 — 标准竞赛模式：81 维 = 60 + 21**
- 基础本体感知 60 维（在 54 维基础上增加 base_height_normalized:1, base_height_error:1, foot_contacts:4）
- 竞赛特定 21 维：
  - `smiley_relative_pos`：3 个笑脸的相对位置（6 维）
  - `hongbao_relative_pos`：3 个红包的相对位置（6 维）
  - `trigger_flags`：6 个触发标志
  - `finish_relative_pos`：终点位置（2 维）
  - `finish_flag`：终点到达标志（1 维）

**模式三 — 崎岖地形模式（`rough_terrain_mode=True`）：54 + 17 + N×27 维**
- 基础 54 维 + 地形感知 17 维（foot_contacts:4 + foot_forces_body:12 + base_height:1）
- 状态历史 N 帧 × 27 维/帧（joint_pos_rel:12 + joint_vel:12 + gravity:3）
- 默认 N=3，总计 54 + 17 + 81 = **152 维**

> **互斥校验：** 课程模式与崎岖地形模式不可同时启用（代码中会抛出 `ValueError`）。

#### 1.3.4 奖励设计细节

| 奖励组件 | 权重 | 说明 |
|---------|------|------|
| 存活奖励 | +0.01（站立）/ -0.02（趴下） | 高度门控生存信号 |
| 站立高度 | 2.0 × exp(-err²/0.05) | 高斯核形式 |
| 前进速度 | 1.0 × clip(v, -0.5, 2.0) | 朝目标速度投影 |
| 路径进度 | clip(Δd × 0.5, -0.1, 0.5) | 距离缩减奖励 |
| 终止惩罚 | **-50.0** | 宽容化，鼓励探索 |

**竞赛计分要素（一次性稀疏奖励）：**

| 计分项 | 分值 | 触发条件 |
|-------|------|---------|
| 笑脸触发 × 3 | +4.0 × 3 = **12 分** | 进入触发半径 1.0m |
| 红包收集 × 3 | +2.0 × 3 = **6 分** | 进入触发半径 1.0m |
| 终点到达 | **+20 分** | 进入终点区域 |
| 庆祝完成 | **+2 分** | 停稳后自动注入庆祝序列 |

**庆祝动作自动注入机制：**
到达终点后，`apply_action` 中自动注入预设的庆祝运动序列：交替抬腿 + 身体摇摆 + 小跳，频率 3Hz。该机制确保评估时能可靠触发庆祝加分。

### 1.4 第三赛段训练策略（Section 012 — 复杂地形赛道）

#### 1.4.1 整体训练思路

Section 012 是**难度最大、得分项最丰富**的赛段。机器狗从"2026"平台出发，穿越波浪地形、选择吊桥或河床路线、收集红包、下楼梯，最终到达"丙午大吉"终点平台。

本赛段是整个方案**奖励工程优化最密集**的环节，经历了多轮迭代调优（标记为 Fix#6 ~ Fix#17），核心突破如下：

1. **消除安全港效应（Fix#16）：** HEIGHT_REWARD_SCALE 从 2.0 降至 0.5，消除"原地站高就赚分"的局部最优
2. **拓宽梯度盆（Fix#17）：** VEL_TRACKING_SIGMA 从 0.25 拓宽至 2.0，确保低速时仍有梯度引导
3. **强化前进激励（Fix#16）：** FORWARD_VEL_SCALE 提升至 3.0，PROGRESS_SCALE 提升至 4.0
4. **宽容化探索（Fix#16）：** TERMINATION_PENALTY 从 -200 降至 -50，降低保守倾向

**赛道路线设计：**

```
"2026"平台 (Y≈7.83) → 波浪地形 (Y>12) → 路线分叉 (Y>14.5)
                                            ├─ 吊桥路线 (X<-1.0) ← 难度高, +10分
                                            └─ 河床路线 (X>0.0) ← 难度低, +5分
                                                → 汇合 → 下楼梯 (Y>23) → "丙午大吉"终点 (Y≈24.33)
```

#### 1.4.2 终止条件

| 终止条件 | 阈值 | 说明 |

| 基座接触地面 | `base_contact` 传感器触发 | 躯干碰地 |
| 姿态失控 | Roll/Pitch > 45° | 与 011 一致 |
| 越界 | X: ±5.5, Y: 2.0 ~ 26.0 | 赛道矩形边界 |
| 高度跌落 | < 40% × 目标高度 | 比例式判定 |
| 关节异常 | >100 或 NaN/Inf | 仿真保护 |
| 宽限期 | 前 10 步 | 重置宽限 |

#### 1.4.3 数据处理与观测量

**标准模式：54 维**（与 001 完全一致的观测结构）

### 1.5 第四赛段训练策略（Section 013 — 滚球+随机地形）

#### 1.5.1 整体训练思路

Section 013 是**抗冲击鲁棒性**的终极考验。机器狗需穿越 3 个滚动金球障碍区，跨越随机地形（Random Terrain），最终抵达"中国结"终点完成庆祝。

本赛段的核心策略创新在于**引导模型选择高分"策略B"**（触碰金球但保持不摔倒 +15分），而非保守的"策略A"（绕行避障 +10分）。通过设定 `BALL_SURVIVE_SHAPING = 2.0` 的即时奖励，鼓励机器狗学会高阶的抗冲击鲁棒动作。

**终止惩罚回调至 -200：** 在此赛段，保持站立的重要性远超探索，因此恢复重罚以严守生存底线。

**赛道结构：**

```
起点 (Y≈26.0) → 3个滚动金球 (Y≈31.23)
                   ├─ 策略A: 避障通过 → +10分
                   └─ 策略B: 碰球存活 → +15分（互斥）
                → 随机地形 (Y>30.5) → +5分
                → "中国结"终点 (Y≈32.3) → +5分
                → 庆祝动作 → +5分
```

#### 1.5.2 终止条件

| 终止条件 | 阈值 | 说明 |

| 姿态失控 | Roll/Pitch > 45° | |

| 关节异常 | >100 或 NaN/Inf | |
| Episode 超时 | ≥ max_episode_steps | Section 013 独有，由本类直接处理 |

> **注意：** Section 013 的 X 轴边界收紧至 ±4.9（比 012 的 ±5.5 更窄），越界即触发 -200 重罚，强制模型学习**"边界生存"**本能。

#### 1.5.3 数据处理与观测量

观测空间结构与 Section 012 完全一致：

#### 1.5.5 金球策略抉择

本赛段最具策略深度的设计在于**互斥评分机制**：

| 策略 | 条件 | 分值 | 难度 |
|------|------|------|------|
| 策略A（避障优先） | Y > 31.5 **且** 未碰任何球 | **+10** | 低 |

- 三个金球位置：`(3, 31.23)`, `(0, 31.23)`, `(-3, 31.23)`
- 接触判定半径：**1.5m**
- 碰触后仍站立 → 触发 `BALL_SURVIVE_SHAPING = 2.0` 即时奖励

我们的模型瞄准**策略B**训练：通过 `BALL_SURVIVE_SHAPING` 即时奖励引导策略网络学会碰球后利用强大的鲁棒控制力维持平衡，从而获取更高的 15 分奖励。

**防失败兜底机制：**

| 机制 | 设计 | 效果 |
|------|------|------|
| **宽限期** | `GRACE_PERIOD_STEPS = 10` | 避免重置瞬间数值突变误判摔倒 |
| **动态终止惩罚** | 011/012: -50, 013: -200 | 平衡探索与生存约束 |
| **倾斜角阈值** | 001: 30°, 011~013: 45° | 按地形复杂度差异化 |
| **动作平滑滤波** | α=0.3 一阶低通 | 抑制关节高频抖动 |
| **力矩限幅** | [17, 17, 34] × 4 腿 | 防止电机过载 |
| **观测噪声注入** | 关节速度噪声1.5，陀螺仪0.2 | 提升抗干扰鲁棒性 |
| **NaN/Inf 检测** | 每步检测关节状态 | 物理仿真异常即时终止 |

> 上述防失败机制确保模型具备极高的评测成功率：即使在极小概率出现的"穿模"或传感器数值突变情况下，宽限期缓冲机制也能给予模型自我恢复的机会，显著降低误判概率。
